# 福井中文社交媒体数据抓取手动运行指南

本文档面向不熟悉编程的 Windows 用户，默认使用 **VS Code + WSL Ubuntu** 运行本项目。请按顺序操作，不要跳过“备份与安全规则”部分。

## 1. 这个项目会生成哪些文件

最常用的福井小红书流程会生成两个 CSV 文件：

- `fukui_xhs_reviews.csv`：原始抓取结果，每一行是一条小红书笔记搜索结果。
- `fukui_xhs_analysis.csv`：分析后的分类结果，会增加 `theme`、`fan_score`、`travel_score` 等列。

程序现在默认不会直接覆盖旧数据：

- 每次写入已有 CSV 前，会先在 `backups/` 文件夹里保存一个带时间戳的备份。
- 默认会把新抓到的数据和旧 CSV 合并，而不是删除旧数据。
- 如果这次抓取结果为空，程序默认拒绝写入，避免把旧数据清空。
- 如果新结果比旧结果少很多，程序默认拒绝“缩小式覆盖”。

只有在非常确定的情况下，才使用 `--replace --allow-shrink` 这类危险参数。

## 2. 推荐工作方式：VS Code + WSL Ubuntu

### 第一次准备

1. 打开 Windows。
2. 打开 VS Code。
3. 在 VS Code 左下角确认已经连接到 WSL Ubuntu。通常左下角会显示类似 `WSL: Ubuntu`。
4. 用 VS Code 打开这个项目文件夹。
5. 在 VS Code 顶部菜单选择 `Terminal` -> `New Terminal`。
6. 确认终端里显示的是 Ubuntu/Linux，不是 PowerShell。可以输入：

```bash
pwd
```

如果路径类似 `/home/...` 或 `/mnt/c/...`，一般就是 WSL 终端。

### 安装 Python 依赖

第一次运行前，在 VS Code 的 WSL 终端里执行：

```bash
python3 -m pip install -r requirements.txt
python3 -m playwright install chromium
```

如果提示 `pip` 不存在，先执行：

```bash
sudo apt update
sudo apt install -y python3-pip
```

然后再重新执行上面的安装命令。

## 3. 运行前如何准备浏览器登录

抓取小红书或抖音需要使用你自己的账号登录。请注意：脚本会打开一个 Playwright 控制的 Chromium 窗口，这个窗口不一定等同于你平时使用的 Chrome。

推荐做法：

1. 先手动打开小红书网页版，确认自己的账号可以正常登录：

```text
https://www.xiaohongshu.com
```

2. 然后运行抓取脚本。脚本会弹出 Chromium 浏览器窗口。
3. 如果弹出的窗口要求登录，请在这个弹出的窗口里登录自己的小红书账号。
4. 登录完成后，如果页面没有继续加载搜索结果，可以关闭脚本并重新运行一次。
5. 抓取过程中不要关闭弹出的 Chromium 窗口。
6. 不建议使用 `--headless`，因为无界面模式下无法手动登录。

如果 WSL Ubuntu 不能弹出浏览器窗口，通常是 WSL 图形界面没有配置好。更简单的解决方法是使用 Windows 版 Python 在 PowerShell 里运行同样命令，或者请熟悉电脑的人帮忙确认 WSLg 是否可用。

## 4. 最简单的一条命令：运行完整福井流程

安装好依赖并确认可以登录小红书后，推荐直接运行：

```bash
python3 run_fukui_pipeline.py
```

这条命令会自动执行两步：

1. 抓取小红书中关键词 `福井` 的搜索结果，保存到 `fukui_xhs_reviews.csv`。
2. 分析 `fukui_xhs_reviews.csv`，保存到 `fukui_xhs_analysis.csv`。

如果想多滚动几次，抓取更多结果：

```bash
python3 run_fukui_pipeline.py --scrolls 30
```

如果已经抓取过数据，只想重新运行分析：

```bash
python3 run_fukui_pipeline.py --skip-pull
```

## 5. 分步骤运行

### 只抓取福井小红书数据

```bash
python3 xhs_fukui_reviews.py --output fukui_xhs_reviews.csv
```

增加滚动次数：

```bash
python3 xhs_fukui_reviews.py --output fukui_xhs_reviews.csv --scrolls 30
```

更换搜索关键词，例如 `福井旅游`：

```bash
python3 xhs_fukui_reviews.py --keyword "福井旅游" --output fukui_xhs_reviews.csv
```

### 只分析已有福井数据

```bash
python3 xhs_fukui_analysis.py --input fukui_xhs_reviews.csv --output fukui_xhs_analysis.csv
```

### 抓取金泽小红书对照数据

```bash
python3 xhs_kanazawa_reviews.py --output kanazawa_xhs_reviews.csv
```

### 抓取金泽抖音对照数据

```bash
python3 douyin_kanazawa_reviews.py --output kanazawa_douyin_reviews.csv
```

## 6. 数据安全规则

为了避免再次发生数据丢失，请遵守以下规则：

1. 每次正式抓取前，先确认当前文件夹里已有的 CSV 文件是否还在。
2. 不要手动删除 `fukui_xhs_reviews.csv`、`fukui_xhs_analysis.csv` 或 `backups/` 文件夹。
3. 正常运行时不要使用 `--replace`。
4. 如果程序提示拒绝写入空数据，这是保护机制，不是错误。通常说明没有登录成功、网页结构变了、网络失败，或搜索结果没有加载出来。
5. 如果程序提示拒绝把旧数据替换成更少的行数，不要立刻强制覆盖。先检查新数据是否真的完整。
6. 每次写入旧文件前，程序会自动生成备份，位置类似：

```text
backups/fukui_xhs_reviews.csv.20260611T120000Z.bak
```

如果误操作，可以从 `backups/` 找回之前的数据。

## 7. 危险参数：只有确认后才使用

正常研究流程不需要下面这些参数。

如果你确实想完全替换旧 CSV：

```bash
python3 xhs_fukui_reviews.py --replace --allow-shrink --output fukui_xhs_reviews.csv
```

如果你确实想允许写入空文件：

```bash
python3 xhs_fukui_reviews.py --allow-empty --output fukui_xhs_reviews.csv
```

除非你非常确定，否则不要使用这些命令。

## 8. 常见问题

### 问题：浏览器没有弹出来

可能原因是 WSL Ubuntu 的图形界面不可用。可以先确认自己是否使用 Windows 11 的 WSLg。也可以改用 Windows PowerShell 和 Windows 版 Python 运行。

### 问题：显示没有找到任何笔记

常见原因：

- 没有在弹出的 Chromium 窗口里登录小红书。
- 页面加载太慢。
- 小红书要求验证。
- 网页结构发生变化，脚本需要更新。

可以先不要删除任何 CSV，重新登录后再运行：

```bash
python3 run_fukui_pipeline.py
```

### 问题：程序拒绝写入，因为结果为空

这是正常保护机制。旧数据没有被覆盖。请检查浏览器登录状态和网络。

### 问题：程序生成了很多备份文件

这是正常的。备份文件是为了防止学术数据丢失。确认最终数据没有问题后，可以把备份文件复制到安全位置长期保存。

### 问题：怎样确认结果文件有数据

可以运行：

```bash
python3 - <<'PY'
import csv
for path in ["fukui_xhs_reviews.csv", "fukui_xhs_analysis.csv"]:
    try:
        with open(path, encoding="utf-8", newline="") as f:
            rows = list(csv.DictReader(f))
        print(path, "行数:", len(rows))
    except FileNotFoundError:
        print(path, "不存在")
PY
```

## 9. 推荐的正式抓取流程

每次正式抓取建议按这个顺序：

1. 打开 VS Code。
2. 进入 WSL Ubuntu 终端。
3. 确认项目文件夹正确。
4. 执行：

```bash
python3 run_fukui_pipeline.py --scrolls 30
```

5. 如果弹出 Chromium 要求登录，在弹出的浏览器中登录小红书。
6. 如果第一次登录后没有抓到数据，重新执行同一条命令。
7. 看到 `Saved ... total rows` 和 `Classification output saved` 后，说明流程完成。
8. 不要删除 CSV 和 `backups/`。
